# 🤖 AI Telegram Bot (Groq API)

Асинхронный Telegram-бот на базе `aiogram 3` и `openai 2`, использующий сверхбыстрый инференс **Groq**. Поддерживает текстовый диалог с памятью, анализ изображений (Vision) и генерацию аудио (TTS).

> ⚡ **Почему Groq?** Мгновенные ответы LLM, щедрые бесплатные лимиты для разработчиков и полная совместимость с OpenAI SDK.

## ✨ Возможности

| Режим | Описание | Используемая модель |
|--------|----------|---------------------|
| 💬 **Текст** | Диалог с ИИ с сохранением истории сообщений | `llama-3.3-70b-versatile` |
| 🖼️ **Зрение** | Анализ и детальное описание загруженных фото | `meta-llama/llama-4-scout-17b-16e-instruct` |
| 🎙️ **TTS** | Преобразование текста в голосовое сообщение `.wav` | `canopylabs/orpheus-v1-english` |

## 🚀 Быстрый старт

### 1️⃣ Подготовка окружения
```bash
# Клонирование (если репозиторий уже создан)
git clone <your-repo-url>
# Создание и активация виртуального окружения
python -m venv venv
# Windows:
venv\\Scripts\\activate
# macOS/Linux:
source venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt
```

### 2️⃣ Настройка `.env`
Создайте файл `.env` в корневой директории и добавьте переменные:

```env
# Telegram Bot Token (получить у @BotFather)
BOT_KEY=your_telegram_bot_token

# Groq API Key (создать на https://console.groq.com/keys)
AI_KEY=gsk_your_groq_api_key

# ⚠️ ОБЯЗАТЕЛЬНО: Ссылка на Groq OpenAI-совместимый эндпоинт
BASE_URL=https://api.groq.com/openai/v1
```

> 🔑 **Важно:** `BASE_URL` должен быть именно `https://api.groq.com/openai/v1`. Без него `openai` SDK будет обращаться к стандартному API OpenAI и вернёт ошибку авторизации.

### 3️⃣ Запуск
```bash
python bot.py
```
Бот запустится в режиме `polling` и начнёт обрабатывать сообщения.

## 📖 Команды бота

| Команда | Описание |
|---------|----------|
| `/start`  | Приветствие и справка по режимам |
| `/text`   | Включает режим текстового диалога. Просто пишите вопросы |
| `/image`  | Включает режим анализа фото. Отправьте изображение (можно с подписью) |
| `/audio`  | Включает режим озвучки. Отправьте текст для генерации голоса |

> 💡 После получения ответа бот автоматически сбрасывает состояние и ждёт новую команду.

## ⚙️ Технические детали
- **Фреймворк**: `aiogram 3.27.0` + `openai 2.30.0`
- **Состояния**: `MemoryStorage` (FSM) для изоляции режимов ввода
- **Обработка фото**: Автоматическая загрузка с ретраями (до 3 попыток, таймаут 30с)
- **Формат Vision**: Изображения кодируются в `base64` и передаются как `data:image/jpeg;base64,...`
- **Аудио**: Возвращается `aiogram.types.BufferedInputFile` с расширением `.wav`

### 🧠 Глобальный контекст диалога
В `text_ai.py` история сообщений хранится в общей переменной `messages`. Это удобно для тестов, но для публичного бота рекомендуется привязывать контекст к `user_id` через `FSM` или базу данных.

### 🔒 Безопасность
Никогда не публикуйте `.env` в репозитории. Добавьте строку `.env` в файл `.gitignore`.
